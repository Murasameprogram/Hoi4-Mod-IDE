# HOI4 Mod IDE 插件开发规范指南（版本 1.0 简体中文版）

本指南旨在帮助开发者创建符合 HOI4 Mod IDE 插件系统的扩展插件。由于当前插件系统采用**独立加载模式**，插件与主程序完全隔离，因此插件必须自包含所有依赖，且无法直接调用主程序服务。请仔细阅读以下规范，确保您的插件能够正确加载和运行。

---

## 1. 插件系统概述与当前状态

HOI4 Mod IDE 的插件系统基于 `McMaster.NETCore.Plugins` 库实现，允许通过放置 DLL 文件在 `Plugins` 文件夹来扩展 IDE 功能。

### 当前状态（版本 1.0）
- **加载模式**：每个插件在独立的 `AssemblyLoadContext` 中加载，与主程序类型隔离。
- **`sharedTypes` 配置**：当前设置为空数组，意味着插件使用的所有类型（包括 `IPlugin` 接口）均来自插件自身携带的程序集，而非主程序。
- **服务访问**：插件**不能**通过 `IServiceProvider` 获取主程序提供的服务（如导航、日志等），因为接口类型不相等。
- **依赖要求**：插件必须携带所有运行所需的程序集（包括 `PluginContracts.dll` 及其所有 NuGet 依赖）。

未来版本可能引入共享类型机制，允许插件调用主程序服务，届时本指南将更新。但现有插件仍可继续运行，只需遵循当前规范。

---

## 2. 插件开发环境准备

- **Visual Studio 2022**（或更高版本），支持 .NET 9。
- **.NET 9 SDK**（必须安装，与主程序目标框架一致）。
- **HOI4 Mod IDE 主程序**（用于测试插件加载）。
- **PluginContracts 项目**（包含 `IPlugin` 接口定义，需要引用）。

建议将插件项目与主程序解决方案放在一起，方便引用和调试。

## 3. 插件项目配置

### 3.1 创建类库项目
在解决方案中新建一个 **WPF类库/类库** 项目，目标框架选择 **.NET 9.0 (Windows)**。

### 3.2 编辑项目文件（.csproj）
以下是一个完整的项目文件示例，包含必要的配置和详细注释。

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <!-- 目标框架必须为 net9.0-windows，以支持 WPF 并匹配主程序 -->
    <TargetFramework>net9.0-windows</TargetFramework>
    
    <!-- 启用可为空引用类型（可选，推荐） -->
    <Nullable>enable</Nullable>
    
    <!-- 启用 WPF 支持，因为插件界面使用 WPF -->
    <UseWPF>true</UseWPF>
    
    <!-- 关键设置：将项目的所有依赖项（包括传递依赖）复制到输出目录 -->
    <CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>
    
    <!-- 可选：输出文件不生成版本号后缀，保持文件名简洁 -->
    <AppendTargetFrameworkToOutputPath>false</AppendTargetFrameworkToOutputPath>
  </PropertyGroup>

  <ItemGroup>
    <!-- 引用 PluginContracts 项目，必须添加 -->
    <ProjectReference Include="..\PluginContracts\PluginContracts.csproj" />
  </ItemGroup>

  <!-- 可选：生成后事件，自动将插件文件复制到主程序的 Plugins 目录（根据实际路径调整） -->
  <Target Name="CopyToPlugins" AfterTargets="Build">
    <!-- 定义源文件集合：输出目录下所有文件 -->
    <ItemGroup>
      <PluginFiles Include="$(OutputPath)\**\*.*" />
    </ItemGroup>
    <!-- 目标路径：主程序输出目录下的 Plugins\$(AssemblyName) 文件夹 -->
    <Copy SourceFiles="@(PluginFiles)"
          DestinationFolder="..\Hearts of Iron IV Mod IDE\bin\Debug\net9.0-windows\Plugins\$(AssemblyName)\%(RecursiveDir)"
          SkipUnchangedFiles="true" />
  </Target>

</Project>
```

**配置说明**：
- `<CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>` 至关重要，它确保所有 NuGet 包的依赖项都被复制到输出目录，避免运行时缺失。
- 生成后事件将插件文件自动部署到主程序的 `Plugins` 文件夹，方便测试。请根据您的实际项目路径修改 `DestinationFolder`。

## 4. 实现 `IPlugin` 接口

所有插件必须实现 `PluginContracts.IPlugin` 接口。该接口定义在 `PluginContracts` 项目中，需添加对该项目的引用。

### 4.1 接口定义（位于 `PluginContracts` 项目）
```csharp
using System;
using System.Threading.Tasks;

namespace PluginContracts
{
    /// <summary>
    /// 所有插件必须实现的主接口。
    /// </summary>
    public interface IPlugin
    {
        /// <summary>
        /// 获取插件的唯一标识符。建议使用反向域名格式，如 "com.example.myplugin"。
        /// 该标识符将用于区分不同插件，并应与 plugin.json 中的 id 字段一致。
        /// </summary>
        string Id { get; }

        /// <summary>
        /// 获取插件的显示名称。
        /// </summary>
        string Name { get; }

        /// <summary>
        /// 获取插件版本号。推荐遵循语义化版本规范，如 "1.0.0"。
        /// </summary>
        string Version { get; }

        /// <summary>
        /// 获取作者名称。
        /// </summary>
        string Author { get; }

        /// <summary>
        /// 获取插件的简短描述。
        /// </summary>
        string Description { get; }

        /// <summary>
        /// 初始化插件。此方法在插件加载后立即调用。
        /// 注意：由于当前 sharedTypes 为空，hostServices 参数无法用于获取主程序服务。
        /// 插件应在此处执行自身的初始化逻辑（如读取配置、创建资源等）。
        /// </summary>
        /// <param name="hostServices">主程序的服务提供器（当前无用）</param>
        /// <returns>表示初始化完成的 Task</returns>
        Task InitializeAsync(IServiceProvider hostServices);

        /// <summary>
        /// 返回插件的主视图。该视图将在插件管理页面双击卡片时显示。
        /// </summary>
        /// <returns>必须返回一个 System.Windows.FrameworkElement 派生对象（如 UserControl、Page、Grid 等）</returns>
        object GetMainView();
    }
}
```

### 4.2 插件实现示例
创建一个类 `Plugin.cs`，内容如下：

```csharp
using System;
using System.Threading.Tasks;
using System.Windows;
using System.Windows.Controls;
using PluginContracts;

namespace MyPlugin
{
    /// <summary>
    /// 插件主类，实现 IPlugin 接口。
    /// </summary>
    public class Plugin : IPlugin
    {
        /// <summary>
        /// 插件唯一标识符，应与 plugin.json 中的 id 保持一致。
        /// </summary>
        public string Id => "com.example.myplugin";

        /// <summary>
        /// 插件显示名称。
        /// </summary>
        public string Name => "我的插件";

        /// <summary>
        /// 插件版本号。
        /// </summary>
        public string Version => "1.0.0";

        /// <summary>
        /// 作者信息。
        /// </summary>
        public string Author => "开发者姓名";

        /// <summary>
        /// 插件简短描述。
        /// </summary>
        public string Description => "这是一个示例插件，用于演示插件开发。";

        /// <summary>
        /// 初始化方法。此处可进行资源加载、配置读取等操作。
        /// 注意：当前版本无法使用主程序服务，hostServices 参数保留以备未来扩展。
        /// </summary>
        public Task InitializeAsync(IServiceProvider hostServices)
        {
            // 示例：初始化日志（自行实现文件日志）
            // Log("插件初始化...");
            return Task.CompletedTask;
        }

        /// <summary>
        /// 返回插件的主视图。此方法将在 UI 线程上调用，因此可直接创建 WPF 控件。
        /// </summary>
        public object GetMainView()
        {
            // 创建一个简单的用户控件作为主视图
            var view = new MyPluginMainView();
            return view;
        }
    }

    /// <summary>
    /// 插件主视图（用户控件）。
    /// </summary>
    public partial class MyPluginMainView : UserControl
    {
        public MyPluginMainView()
        {
            // 可以在构造函数中初始化 UI 元素，但推荐使用 XAML 定义界面。
            // 这里以代码方式简单创建，实际开发中建议使用 XAML 文件。
            var stackPanel = new StackPanel
            {
                Margin = new Thickness(20),
                HorizontalAlignment = HorizontalAlignment.Center,
                VerticalAlignment = VerticalAlignment.Center
            };

            var textBlock = new TextBlock
            {
                Text = "欢迎使用我的插件！",
                FontSize = 18,
                FontWeight = FontWeights.Bold,
                Margin = new Thickness(0, 0, 0, 10)
            };
            stackPanel.Children.Add(textBlock);

            var button = new Button
            {
                Content = "点击测试",
                Width = 100,
                Height = 30
            };
            button.Click += (s, e) =>
            {
                MessageBox.Show("插件功能正常！", "提示");
            };
            stackPanel.Children.Add(button);

            this.Content = stackPanel;
        }
    }
}
```

**注意**：更规范的做法是使用 XAML 定义用户控件。创建 `MyPluginMainView.xaml` 和 `MyPluginMainView.xaml.cs`，然后在 `GetMainView()` 中返回该控件实例即可。

## 5. 插件清单文件 `plugin.json`

每个插件文件夹的根目录下必须包含 `plugin.json` 文件，用于描述插件的元数据和入口点。文件格式如下（带注释的 JSON 不合法，此处仅为说明，实际应删除注释）：

```json
{
  "id": "com.example.myplugin",      // 插件唯一标识，必须与 IPlugin.Id 一致
  "name": "我的插件",                 // 显示名称
  "version": "1.0.0",                // 版本号
  "author": "开发者姓名",              // 作者
  "description": "这是一个示例插件",    // 描述
  "entryPoint": "MyPlugin.dll",       // 主程序集文件名（包含 .dll 扩展名）
  "minHostVersion": "1.0.0"           // 可选，所需最低主程序版本（当前未强制校验）
}
```

**重要注意事项**：
- 文件必须保存为 **UTF-8 无 BOM** 编码，否则可能导致 JSON 解析失败。
- `entryPoint` 必须与编译输出的主程序集文件名完全一致。
- 建议将 `plugin.json` 文件属性设置为“如果较新则复制”，以便在编译时自动复制到输出目录。

在项目文件中添加以下内容以自动复制 `plugin.json` 到输出目录：
```xml
<ItemGroup>
  <None Update="plugin.json">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </None>
</ItemGroup>
```

## 6. 依赖项处理与部署

由于 `sharedTypes` 为空，插件必须携带所有运行所需的程序集。部署时请遵循以下步骤：

### 6.1 编译插件
- 在 Visual Studio 中生成插件项目，确保生成成功且所有依赖项已被复制到输出目录（如 `bin\Debug\net9.0-windows\`）。

### 6.2 手动部署
1. 找到主程序的输出目录（例如 `...\Hearts of Iron IV Mod IDE\bin\Debug\net9.0-windows\`）。
2. 在该目录下创建 `Plugins` 文件夹（如果不存在）。
3. 在 `Plugins` 文件夹内创建一个以插件标识符命名的子文件夹（例如 `Plugins\com.example.myplugin\`）。
4. 将插件输出目录下的**所有文件**（包括 `.dll`、`.pdb`、`.json` 及其他资源文件）复制到该子文件夹。

### 6.3 自动部署（推荐）
在插件项目的 `.csproj` 中添加生成后事件（如前文示例），这样每次编译后会自动将文件复制到主程序的 `Plugins` 目录下。这可以大大简化测试流程。

**示例生成后事件（已包含在项目文件示例中）**：
```xml
<Target Name="CopyToPlugins" AfterTargets="Build">
  <ItemGroup>
    <PluginFiles Include="$(OutputPath)\**\*.*" />
  </ItemGroup>
  <Copy SourceFiles="@(PluginFiles)"
        DestinationFolder="..\Hearts of Iron IV Mod IDE\bin\Debug\net9.0-windows\Plugins\$(AssemblyName)\%(RecursiveDir)"
        SkipUnchangedFiles="true" />
</Target>
```
请根据您的实际项目路径调整 `DestinationFolder`。如果主程序输出路径在不同配置下不同（如 Release），可考虑使用条件判断。

### 6.4 依赖项清单
确保插件文件夹包含以下必要文件：
- `YourPlugin.dll`（插件主程序集）
- `PluginContracts.dll`（接口程序集）
- 所有 NuGet 包依赖的 `.dll`（通过 `CopyLocalLockFileAssemblies` 自动复制）
- `plugin.json`

**不要遗漏任何 `.dll`**，否则插件加载时可能会因缺少依赖而失败。

## 7. UI 开发指南

### 7.1 视图返回类型
`GetMainView()` 必须返回一个 `FrameworkElement` 派生对象，可以是：
- `UserControl`（推荐，便于 XAML 设计）
- `Page`
- 任何 `Panel`（如 `Grid`、`StackPanel`）
- 自定义控件

返回的控件将被放置在一个独立的新窗口中显示，窗口标题默认设置为插件名称。

### 7.2 使用 XAML
建议为插件主视图创建 WPF 用户控件（`.xaml` 和 `.xaml.cs`），以便利用设计器和数据绑定。

**示例：`MyPluginMainView.xaml`**
```xml
<UserControl x:Class="MyPlugin.MyPluginMainView"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             Width="400" Height="300">
    <Grid Margin="20">
        <Grid.RowDefinitions>
            <RowDefinition Height="Auto"/>
            <RowDefinition Height="Auto"/>
        </Grid.RowDefinitions>
        <TextBlock Grid.Row="0" Text="欢迎使用我的插件！" 
                   FontSize="18" FontWeight="Bold" HorizontalAlignment="Center"/>
        <Button Grid.Row="1" Content="点击测试" Width="100" Height="30" 
                HorizontalAlignment="Center" Margin="0,20,0,0" Click="Button_Click"/>
    </Grid>
</UserControl>
```

**代码隐藏：`MyPluginMainView.xaml.cs`**
```csharp
using System.Windows;
using System.Windows.Controls;

namespace MyPlugin
{
    public partial class MyPluginMainView : UserControl
    {
        public MyPluginMainView()
        {
            InitializeComponent();
        }

        private void Button_Click(object sender, RoutedEventArgs e)
        {
            MessageBox.Show("插件功能正常！", "提示");
        }
    }
}
```

然后在 `Plugin.cs` 中返回该控件实例：
```csharp
public object GetMainView()
{
    return new MyPluginMainView();
}
```

### 7.3 资源管理
- 插件可以定义自己的资源字典（如样式、画笔），建议将资源放在控件级别或应用级别，但不要假设主程序会提供任何特定资源。
- 如果使用 `DynamicResource`，请确保资源在插件内部定义，否则可能导致运行时错误。

### 7.4 窗口行为
- 插件视图显示在一个**可调整大小、拥有所有者（主窗口）的独立窗口**中。用户关闭该窗口后，插件不会卸载，但视图将被销毁。
- 插件不应尝试直接操作主窗口（如修改标题、关闭主窗口等），以免产生不可预知的行为。

## 8. 调试插件

1. **将插件项目和主程序放在同一解决方案**。
2. **设置主程序为启动项目**。
3. **在插件代码中设置断点**（例如在 `InitializeAsync` 或视图构造函数中）。
4. **启动调试（F5）**，主程序运行后会自动加载插件（需确保插件已正确部署到 `Plugins` 文件夹）。
5. 当执行到断点处，调试器应能中断。如果无法命中，请检查：
   - 插件是否已正确部署且被主程序加载（查看插件管理页面）。
   - 符号文件（`.pdb`）是否与插件 DLL 在同一目录。
   - 确保生成配置为 Debug。

**注意**：由于插件运行在独立加载上下文，某些高级调试功能（如“编辑并继续”）可能受限。

## 9. 当前限制与未来兼容性

### 9.1 已知限制
- **无法调用主程序服务**：如导航、日志、获取当前项目信息等。插件若需要这些功能，必须自行实现（例如自行读写文件、实现日志系统）。
- **类型隔离**：插件与主程序之间的类型（包括接口）被视为不同，因此不能直接传递对象。所有交互必须通过基础类型或序列化。
- **插件间通信**：无法直接调用其他插件，但可通过文件、进程间通信（如命名管道）间接实现。

### 9.2 未来兼容性
后续版本可能引入以下改进：
- **共享服务接口**：允许插件通过 `IServiceProvider` 获取主程序服务，但需要将相应接口类型添加到 `sharedTypes`。
- **插件 API 版本化**：通过 `minHostVersion` 字段进行兼容性检查。
- **动态启用/禁用**：可能支持运行时启用或禁用插件（需重启）。

为了确保您的插件能平滑过渡到未来版本，请遵循以下建议：
- 避免依赖任何未在 `IPlugin` 接口中声明的类型。
- 将所有可能变化的逻辑封装在插件内部，尽量减少对主程序特定行为的依赖。
- 保持插件 Id 唯一且不变，以便未来版本进行版本管理。

## 11. 常见问题解答

**Q: 插件加载失败，日志显示“缺少依赖项”，如何解决？**  
A: 检查插件文件夹是否包含所有必需的 `.dll` 文件。确保在项目文件中启用了 `<CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>`，并重新编译、重新复制所有文件。特别留意 `PluginContracts.dll` 是否存在。

**Q: 插件加载成功，但双击卡片后窗口不显示或报错？**  
A: 请确保 `GetMainView()` 返回的对象是 `FrameworkElement` 派生类，并且没有在构造函数中抛出异常。可以在视图构造函数中设置断点调试。

**Q: 如何在插件中使用第三方 NuGet 包？**  
A: 正常通过 NuGet 安装即可，但必须启用 `CopyLocalLockFileAssemblies`，以便所有依赖项被复制到输出目录。部署时需将所有 `.dll` 复制到插件文件夹。

**Q: 插件能否访问主程序的文件或配置？**  
A: 由于类型隔离，无法直接通过 API 访问，但可以通过文件系统间接访问主程序的工作目录（如当前项目路径）。主程序当前未提供任何共享配置，因此需自行约定文件位置。

**Q: 插件能否创建多个窗口或与主程序交互？**  
A: 插件可以自行创建新窗口（通过 `new Window()`），但无法控制主窗口。与主程序的交互目前只能通过文件或外部进程通信，无法直接调用主程序方法。

**Q: 如何确保插件在未来版本中兼容？**  
A: 遵循本指南，避免依赖未公开的主程序类型。关注官方文档更新，当新版本引入共享服务时，可能需要调整插件代码以利用新功能，但现有插件仍应继续运行。
